### 一、登录界面

#### 布局结构

- **背景层**：一张图片（我自己后续填入）
- **主弹窗**：页面中央蓝色半透明弹窗，占屏幕约 60% 宽度：
    
    - 弹窗右侧顶部：标题 “多人实时在线自习室”（白色字体）；
    - 弹窗左侧：白色线条风格的学习元素图标（后续我自己填入）；
    - 弹窗右侧：登录表单区域（白色背景卡片）：
        
        - 输入框 1：左侧带 “用户” 图标，占位符 “用户名”；
        - 输入框 2：左侧带 “锁” 图标，占位符 “密码”；
        - 操作按钮：蓝色 “登录” 按钮（宽度占满表单）；
        - 辅助链接：按钮右侧下方蓝色 “注册” 文字链接。
        
    

#### 交互逻辑

- 输入框支持键盘输入，自动校验格式（如邮箱格式）；
- 点击 “登录” 按钮：触发表单提交，调用后端认证接口；
- 点击 “注册” 链接：跳转到注册表单页（可选阶段，若未实现则弹窗提示 “暂未开放”）；

### 二、注册界面（Registration Page）

#### 布局结构

- **背景层**：与登录界面一致的图片（我后续自己填入）
- **主弹窗**：页面中央蓝色半透明弹窗（尺寸、配色与登录界面弹窗相同）：
    
    - 弹窗右侧顶部：标题 “多人实时在线自习室 - 注册”（白色字体，延续登录页标题样式）；
    - 弹窗左侧：与登录界面一致的白色线条风格学习元素图标（毕业帽、打开的书本、带书本图案的笔记本电脑、咖啡杯、笔筒），确保视觉风格统一；
    - 弹窗右侧：注册表单区域（白色背景卡片，尺寸与登录表单一致）：
        
        1. 输入框 1：左侧带 “用户” 图标，占位符 “设置用户名”；
        2. 输入框 3：左侧带 “锁” 图标，占位符 “设置密码（至少 6 位）”；
        3. 输入框 4：左侧带 “锁” 图标，占位符 “确认密码”；
        4. 操作按钮：蓝色 “注册” 按钮（宽度占满表单，样式与登录按钮一致）；
        5. 辅助链接：按钮下方右侧文字链接 “已有账号？点击登录”。
        
    
#### 交互逻辑

- **输入实时验证**：
    
    - 用户名输入时：实时校验 “长度（2-12 位）”“无特殊字符”，不符合则输入框下方显示红色提示文字（如 “用户名需 2-12 位且不含特殊字符”）；
    - 密码输入时：实时校验 “长度≥6 位”，不符合则提示 “密码至少 6 位”；
    - 确认密码输入时：实时对比与 “设置密码” 的一致性，不一致则提示 “两次密码输入不一致”；
    
- **注册按钮交互**：
    
    - 表单未通过所有验证时，按钮置灰不可点击；
    - 表单验证通过后，按钮变为可点击状态；点击后调用后端注册接口，提交用户信息；
    - 注册成功：弹窗提示 “注册成功！即将跳转至登录页”，3 秒后自动跳转到登录界面；
    
- **链接交互**：点击 “已有账号？点击登录” 链接，直接跳转到登录界面。

### 三、主界面

#### 布局结构

- **顶部导航栏**（深色背景，全屏宽度）：
    
    - 左侧：平台名称 “多人实时在线自习室”；
    - 中间：导航按钮组（“房间列表”“创建房间”），当前选中 “房间列表”（文字高亮）；
    - 右侧：用户头像（圆形缩略图）+ 通知铃铛图标（带小红点提示）；
    
- **房间列表区**（导航栏下方，流式布局）：
    
    - 以 “卡片” 为单位，每行排列 4 个卡片，自动适配屏幕宽度；
    - 单个房间卡片包含：
        
        1. 顶部标识：房间状态标签（如 “学习中” 绿色、“未开始” 红色）；
        2. 背景图：自习场景图（教室、图书馆等）；
        3. 底部信息栏：房间名称、“当前人数 / 最大容量”、房间状态文字。
        
    

#### 交互逻辑

- 点击 “创建房间” 按钮：弹出配置弹窗，包含输入项（房间名称、开始时间、结束时间、人数上限）+“确认创建” 按钮；创建成功后自动跳转至该房间页面；
- 点击任意房间卡片：触发 “加入房间” 操作，跳转至对应房间页面；
- 点击 “个人中心” 按钮：跳转至个人中心页面；

### 四、个人中心页面

#### 布局结构

- **左侧侧边栏**（深色背景，固定宽度）：
    
    - 顶部：标题：账户信息；下方：用户基础信息（头像 + 用户名）；
    - 菜单列表：“用户信息”“创建的自习室”“标记的自习室”“学习记录” 等；
    
- **主内容区**（侧边栏右侧，网格布局，分多个模块）：
    
    1. **用户 / 金币系统模块**（顶部行，2 列）：
        
        - 左：“用户系统” 卡片，高度约占页面的四分之一（显示当前金币：36）；
        - 右：“金币系统” 卡片（显示总金币：190.09，带 “获取记录”“用途说明” 文字链接）；
        
    2. **学习数据模块**（中间行，3 列）：
        
        - 左：“总学习时长”（折线 + 柱状图，展示时间趋势）；
        - 中：“每日 / 每周统计”（柱状图，对比番茄钟数量与完成率）；
        - 右：“学习记录”（折线图，展示多日专注时长）；
        
    3. **成长系统模块**（底部行，3 列）：
        
        - 中：“虚拟形象系统”（形象进度条，如 “金色头发：26%”）；
        - 左：“学习计划”（环形进度图，展示计划完成率）；
        - 右：“植物系统”（4 个盆栽图标，展示用户已解锁的植物）。


#### 交互逻辑

- 选中 “自习室”（背景高亮）
- 点击 “金币系统” 的 “获取记录”：弹出列表弹窗，展示金币获取的时间、来源（如 “专注 30 分钟 + 5”）；
- 点击 “学习记录” 的 “多日统计”：切换图表的时间维度（日 / 周 / 月）；
- 点击 “植物系统” 的 “合成” 按钮：消耗金币合成更高等级植物；
- 点击 “虚拟形象系统” 卡片：跳转至形象编辑页（更换角色、装饰）；
- 学习数据实时更新：页面定时拉取最新的学习时长、番茄钟数据。

### 四、房间页面（Room Page，核心页面）

#### 布局结构

- **左侧侧边栏**（固定宽度）：
    
    - 顶部：“房间”“信息” 切换标签；
    - 中间：房间基础数据（当前人数、总专注时长等）；
    - 底部：聊天区域（滚动列表展示历史消息）+ 输入框 +“发送” 按钮；
    
- **中间主区域**（全屏核心区）：
    
    - 场景背景：教室实景（书架、窗户、绿植）；
    - 圆桌布局：页面中央圆形桌子，周围分布用户虚拟形象（卡通人物）；
    - 形象状态：每个虚拟形象上方显示状态标签（“专注中” 绿色、“休息中” 黄色、“离开” 灰色）；
    
- **右侧协作区**（固定宽度）：
    
    - 顶部：“房间协作” 标题；
    - 成员列表：展示房间内所有用户（头像 + 昵称 + 实时状态）；
    - 番茄钟模块：红色番茄造型计时器，显示当前倒计时（如 “32:00”），下方显示当前阶段（“休息中”）+ 状态切换开关；
    - 排名模块：（隐藏在成员列表下方，点击展开）按专注时长排序的房间内排名。
    

#### 交互逻辑

- 虚拟形象状态同步：通过 WebSocket 实时更新（成员切换专注 / 休息 / 离开时，形象标签自动变化）；
- 聊天交互：输入框输入文字后点击 “发送”，消息实时同步到所有成员的聊天区；系统自动推送番茄钟状态消息（如 “番茄钟开始：专注 25 分钟”）；
- 番茄钟控制：仅房主可见 “切换” 按钮，点击后切换 “专注 / 休息” 阶段，所有成员的番茄钟同步更新；
- 房间背景设置：房主点击右上角 “更多” 按钮，可选择预设背景或上传自定义背景；
- 中途退出：点击页面右上角 “关闭” 按钮，弹出提示 “中途退出将扣除 50% 金币奖励”，确认后返回主界面。
